# Product Brief — Catering Ecommerce

## Who this is for
A home-based caterer who currently takes orders via text and spreadsheets. They want to move to a storefront where customers self-serve.

## Core customer flow
1. Browse menu, grouped by category.
2. Pick items, choose variant (portion size), pick production date (respects lead time).
3. Choose delivery or pickup. If delivery, enter address; check zone.
4. Checkout via Stripe.
5. Receive confirmation email; later, "in production" and "ready" updates.
6. Application should have an authentication, preferred a user signup

## Core owner flow
1. Admin dashboard: today's orders, upcoming week, production calendar.
2. Mark orders `in_production` when starting, `ready` when done.
3. Edit menu: add items, deactivate, change prices.
4. See daily aggregate: how many of each thing to prep.
5. Order details per customer with quatity, amount for each items and total amount
6. Prepare a costco shopping list for the ingredients for the ordered items
7. Assume food cost is 40%, create a dashboard showing profit and loss per day, Order summary
8. Order transcation details per customer, cash, paypal, venmo, creditcard, zelle
